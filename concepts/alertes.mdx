Une **alerte** est un signal d'opportunité commerciale extrait d'un document public d'une collectivité. C'est l'unité de travail centrale dans Skult : chaque alerte représente une occasion potentielle de vendre votre offre à une collectivité.

## Ce que contient une alerte

| Élément | Description |
| --- | --- |
| **Titre** | L'intitulé de l'opportunité, généré à partir du document. |
| **Collectivité & ville** | La collectivité concernée et sa localisation. |
| **Score de pertinence** | Une note de 0 à 10 indiquant à quel point l'alerte correspond à votre activité. |
| **Besoin** | Ce que la collectivité cherche à faire. |
| **Opportunité** | La façon dont votre offre peut y répondre. |
| **Verbatim** | L'extrait exact du document officiel, pour vérifier le contexte. |
| **Angle d'approche** | Une suggestion de message d'accroche, à personnaliser. |
| **Date de séance** | La date du conseil qui a voté ou débattu le sujet. |
| **Source** | Le lien vers le document public d'origine. |
| **Contacts** | Les interlocuteurs identifiés au sein de la collectivité (voir [Contacts & enrichissement](/concepts/contacts)). |

<Tip>
  Lisez toujours le **verbatim** avant de contacter une collectivité : il vous permet de citer le document exact dans votre message et de montrer que vous comprenez leur contexte.
</Tip>

## Le score de pertinence

Chaque alerte est notée de **0 à 10**. Plus le score est élevé, plus l'opportunité est précise, mûre et alignée avec votre offre. Le score combine plusieurs critères :

| Critère | Ce qu'il évalue |
| --- | --- |
| **Précision du besoin** | Le document est-il vague ou précis (montant, échéance, périmètre) ? |
| **Preuve décisionnelle** | Y a-t-il une étude, un diagnostic ou un cadrage qui amorce le projet ? |
| **Adéquation à votre offre** | Le projet et la collectivité correspondent-ils à votre cible ? |
| **Maturité / urgence** | Le besoin est-il imminent ou simplement évoqué ? |
| **Contact identifié** | Un interlocuteur est-il mentionné dans le document ? |

À titre indicatif :
