Les termes que vous rencontrerez en utilisant Skult.

| Terme | Définition |
| --- | --- |
| **Alerte** | Un signal d'opportunité commerciale extrait d'un document public d'une collectivité. L'unité de travail centrale dans Skult. Voir [Comprendre une alerte](/concepts/alertes). |
| **Collectivité** | Une commune, une intercommunalité, un département ou une région. |
| **PV (procès-verbal)** | Le compte rendu officiel d'une séance de conseil municipal ou régional. |
| **Délibération** | Une décision formelle votée en conseil. |
| **Verbatim** | L'extrait exact du document officiel à l'origine de l'alerte. |
| **Score de pertinence** | Une note de 0 à 10 indiquant l'adéquation de l'alerte avec votre activité. |
| **Tier** | Le niveau de priorité d'une alerte (1, 2 ou 3). |
| **Statut** | L'étape d'une alerte dans votre processus commercial. Voir [Statuts & workflow](/concepts/statuts). |
| **Contact** | Un interlocuteur identifié au sein de la collectivité. |
| **Enrichissement** | La recherche de l'e-mail ou du téléphone d'un contact. Voir [Contacts & enrichissement](/concepts/contacts). |
| **Angle d'approche** | Une suggestion de message d'accroche pour contacter la collectivité. |
| **Pipeline** | La vue de vos alertes organisées par statut. Voir [Pipeline](/guides/pipeline). |
| **Organisation** | Votre entreprise sur Skult. Vos données sont isolées des autres organisations. |
| **Rôle** | Le niveau d'accès d'un utilisateur : administrateur, manager ou commercial. |
